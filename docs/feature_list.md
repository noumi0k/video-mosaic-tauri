## 1. プロジェクト管理

### 1-1. 新規プロジェクト（動画読み込み時に自動作成）
- **目的**: 1 本の動画ファイルとそれに付随するすべてのマスク情報を 1 つの作業単位（プロジェクト）として管理する。
- **動作**: 動画を開いた時点で `Project` オブジェクトが生成される。動画パス・映像メタデータ（解像度、FPS、フレーム数）・空のトラックリスト・エクスポートプリセットを保持する。
- **UI**: ファイルメニュー「動画を開く（Ctrl+O）」から動画ファイルを選択。
- **出力**: 内部の `Project` オブジェクトが初期化される。

### 1-2. プロジェクト保存（JSON）
- **目的**: 編集途中の状態を JSON ファイルに保存し、後から再開できるようにする。
- **動作**: `ProjectStore.save(project, file_path)` が呼ばれ、動画パス・映像メタ・全トラック情報（キーフレーム含む）・エクスポートプリセットを JSON に直列化する。
- **UI**: ファイルメニュー「保存（Ctrl+S）」。初回保存時はファイル保存ダイアログが開く。
- **出力**: `.mosaic` または `.json` 拡張子のプロジェクトファイル。

### 1-3. プロジェクト読み込み
- **目的**: 保存済みのプロジェクトファイルを開いて編集を再開する。
- **動作**: JSON ファイルを読み込み、`Project` オブジェクトに復元する。動画ファイルのパスを元に映像を再接続する。
- **UI**: ファイルメニュー「プロジェクトを読み込む（Ctrl+Shift+O）」からファイルを選択。
- **出力**: 全トラック・キーフレームが復元された状態でプレビューが表示される。

## 2. 動画読み込み

### 2-1. 動画ファイルを開く
- **目的**: 対象の動画ファイルをアプリケーションに読み込み、編集を開始できる状態にする。
- **動作**:
  1. FFprobe で映像のメタデータ（解像度、FPS、総フレーム数、時間）を取得する。
  2. OpenCV で動画ファイルを開き、フレーム単位でのランダムアクセスを可能にする。
- **対応形式**: FFmpeg が対応するすべての動画形式（MP4、MOV、AVI、MKV など）。
- **UI**: ファイル選択ダイアログ。読み込み完了後、プレビューに最初のフレームが表示される。

### 2-2. 映像メタデータ表示
- **目的**: 現在開いている動画の基本情報をユーザーに提示する。
- **表示内容**: 解像度（幅×高さ）、FPS、総フレーム数、再生時間。
- **UI**: 画面下部のステータスバーに常時表示。

## 3. プレビュー表示

### 3-1. フレームプレビュー
- **目的**: 現在選択中のフレームの映像をリアルタイムで表示する。
- **動作**: フレーム番号に対応する映像フレームを OpenCV で取得し、画面に描画する。マスクオーバーレイ（楕円・多角形）を重ねて表示する。
- **UI**: 中央ペイン（`PreviewCanvas`）に表示。ウィンドウリサイズに合わせてアスペクト比を保ちながら拡大縮小する。

### 3-2. マスクオーバーレイ表示
- **目的**: 各トラックのマスク形状（楕円・多角形）をプレビュー上に可視化する。
- **動作**: 各 `MaskTrack` について現在フレームのキーフレームを補間し、形状を計算して描画する。選択中のトラックは強調表示（カラーハイライト + ハンドル）される。
- **表示形式**: 楕円トラックは楕円形の枠線＋内部半透明塗りつぶし。多角形トラックは多角形の枠線＋内部塗りつぶし。

### 3-3. モザイクプレビュー ON/OFF
- **目的**: 実際の書き出し結果に近い状態（モザイク適用済み）でプレビューを確認できるようにする。
- **動作**: ON 時は `RenderService.render_frame()` でモザイク処理を適用したフレームを表示する。OFF 時は元の映像を表示する。
- **UI**: キーボード `M` キー、またはツールバーボタンでトグル。
- **注意**: モザイクプレビューは処理負荷が高いため、再生中はコマ落ちする場合がある。

### 3-4. 差分オーバーレイ表示
- **目的**: モザイクあり/なしの差分（影響範囲）を視覚的に確認する。
- **UI**: キーボード `Shift+M` でトグル。

### 3-5. モードバッジ表示
- **目的**: 現在の操作モード（移動 / 頂点編集 / 再生中）をプレビュー右上に常時表示し、ユーザーが現在の状態を把握できるようにする。
- **表示種類**: 「移動モード」「頂点編集モード」「再生中」

## 4. 再生・フレーム移動

### 4-1. 再生 / 停止
- **目的**: 動画をリアルタイムで再生・一時停止する。
- **動作**: 指定した FPS（または再生速度倍率）でフレームを順番に表示する。再生中は編集操作が無効になる。
- **UI**: タイムライン下部の再生ボタン、またはキーボード `Space` キー。

### 4-2. 再生速度変更
- **目的**: 動画を低速または高速で確認できるようにする。
- **速度**: 0.25x / 0.5x / 1x / 2x の 4 段階。
- **UI**: タイムライン下部の速度選択コンボボックス。

### 4-3. フレーム単位移動
- **目的**: 1 フレームずつ正確にフレームを進める/戻す。
- **UI**: キーボード `←` / `→`（1 フレーム）、`Shift+←` / `Shift+→`（10 フレーム）。

### 4-4. 先頭・末尾へのジャンプ
- **目的**: 動画の最初または最後のフレームに素早く移動する。
- **UI**: `Home`（先頭）/ `End`（末尾）。

### 4-5. トラック開始・終了フレームへジャンプ
- **目的**: 選択中のトラックが開始・終了するフレームに素早く移動する。
- **UI**: `Shift+Home`（トラック開始）/ `Shift+End`（トラック終了）。

### 4-6. 前後キーフレームへジャンプ
- **目的**: 選択中のトラックの前後のキーフレームに移動する。
- **UI**: `↑` / `↓` キー、または `[` / `]` キー。

### 4-7. In/Out マーカー設定
- **目的**: 書き出しや検出の対象範囲（開始フレーム・終了フレーム）を指定する。
- **UI**: `I`（In マーカー設定）/ `O`（Out マーカー設定）。タイムライン上に赤い三角マーカーで表示。

## 5. タイムライン操作

### 5-1. タイムライン表示
- **目的**: 時系列でフレーム・トラック・キーフレームの配置を視覚的に確認する。
- **構成**:
  - ルーラー（フレーム番号・タイムコード表示、最上段に固定）
  - トラックレーン（各トラックのキーフレームマーカーを帯状に表示）
  - プレイヘッド（現在フレーム位置を赤い縦線で表示）
- **キーフレームマーカーの色**:
  - 金色: auto / interpolated（AI 自動生成・補間）
  - 灰色: predicted（motion 予測補完）
  - 白: manual（ユーザー手動入力）

### 5-2. タイムライン水平ズーム
- **目的**: フレームレベルの詳細な操作ができるよう、タイムラインを拡大縮小する。
- **UI**: `Ctrl + マウスホイール` で拡大縮小。スライダーでも調整可能。

### 5-3. タイムライン水平スクロール
- **目的**: 長い動画の任意の時間帯に移動する。
- **UI**: タイムライン下部のスクロールバー。

### 5-4. タイムライン縦スクロール（トラックレーン）
- **目的**: トラック数が多い場合に、画面外のトラックにアクセスする。
- **動作**: ルーラーは固定のまま、トラックレーン部分のみスクロールする。選択中のトラックは自動的にスクロールして見えるようになる。

### 5-5. キーフレームマーカー凡例
- **目的**: キーフレームの色の意味をユーザーが常に確認できるようにする。
- **UI**: タイムライン端に常時表示。

## 6. トラック管理

### 6-1. 手動トラック作成（楕円）
- **目的**: AI 検出を使わず、ユーザーが任意の位置に楕円形のマスクトラックを作成する。
- **動作**: 現在フレームの中心付近に初期サイズの楕円トラックを生成する。作成後はすぐに移動・スケール編集が可能。
- **UI**: キーボード `N`、またはメニューから「楕円トラック追加」。
- **注意**: 作成時にカテゴリ選択ポップアップが表示される。

### 6-2. 手動トラック作成（多角形）
- **目的**: 複雑な形状のマスクを多角形で作成する。
- **動作**: 初期状態は矩形多角形として生成され、頂点編集で形状を調整する。
- **UI**: キーボード `Shift+N`、またはメニューから「多角形トラック追加」。

### 6-3. トラック選択
- **目的**: 編集対象のトラックを選択する。
- **動作**: 左ペインのトラックリストクリック、またはプレビュー上でマスク形状をクリックすることで選択できる。
- **UI**: 選択中トラックは強調表示（ハイライト + ハンドル）される。

### 6-4. トラック表示 / 非表示
- **目的**: 特定のトラックをプレビューとタイムラインに表示しないようにする。
- **動作**: `visible` フラグを切り替える。非表示トラックは書き出し設定とは独立して制御できる。
- **UI**: キーボード `H`、またはトラックリストの目アイコンをクリック。

### 6-5. トラック削除
- **目的**: 不要なトラックを削除する。
- **動作**: `ProjectEditService.delete_track()` でトラックを削除する。操作は Undo 可能。
- **UI**: キーボード `Delete`、またはトラックリストの右クリックメニュー。

### 6-6. トラック複製
- **目的**: 既存トラックと同じ形状・スタイルのトラックを別 ID で複製する。
- **動作**: 全キーフレームをコピーして新しい `MaskTrack` を生成する。
- **UI**: トラックリスト右クリックメニュー「複製」。

### 6-7. トラック分割
- **目的**: 1 本のトラックを現在フレームで 2 本に分割する。前半と後半が独立したトラックになる。
- **動作**: 現在フレームを境界として前半・後半のキーフレームをそれぞれ新トラックに分配する。
- **UI**: トラックリスト右クリックメニュー「分割」。

### 6-8. トラック書き出し対象 ON/OFF
- **目的**: 書き出し時に特定のトラックのモザイクを適用しないようにする（表示はされるが書き出し対象外）。
- **動作**: `export_enabled` フラグを切り替える。対象外トラックはプレビューでドット線・半透明・グレーで表示される。
- **UI**: トラックリストの「書き出し」チェックボックス。

### 6-9. トラック一覧パネル
- **目的**: 全トラックの一覧と状態（active/lost/inactive）、ユーザーロック状態を視覚的に管理する。
- **表示情報**: トラック ID、ラベル（カテゴリ名）、状態、ユーザーロックアイコン、visible アイコン、export_enabled チェック。

## 7. マスク編集（プレビュー上）

### 7-1. マスク移動
- **目的**: マスクの位置を変更する。
- **動作**: マスク内部をドラッグすると中心位置が移動する。移動はキーフレームとして記録される（manual source）。
- **UI**: マスク形状内部をドラッグ。

### 7-2. マスクスケール（等倍）
- **目的**: マスクを縦横比を保ったまま拡大縮小する。
- **UI**: `Shift + ドラッグ`（ハンドルを操作）。

### 7-3. マスクスケール（独立）
- **目的**: マスクの幅または高さを独立して変更する。
- **UI**: `Ctrl + ドラッグ`（ハンドルを操作）。

### 7-4. マスク回転（楕円）
- **目的**: 楕円マスクの傾きを調整する。
- **動作**: 回転角度を 0〜360° で設定する。キーフレームに回転値が保存される。
- **UI**: `Shift + Ctrl + ドラッグ`。

### 7-5. 多角形頂点の移動
- **目的**: 多角形マスクの各頂点を個別に移動して形状を細かく調整する。
- **UI**: 頂点編集モードで頂点をドラッグ。

### 7-6. 頂点追加
- **目的**: 多角形に新しい頂点を追加して、形状を増やす。
- **UI**: 辺（エッジ）をダブルクリック、または右クリックメニュー「頂点を追加」。

### 7-7. 頂点削除
- **目的**: 不要な頂点を削除してシンプルな形状にする。
- **UI**: 頂点を `Alt + 右クリック`、または右クリックメニュー「頂点を削除」。

## 8. キーフレーム操作

### 8-1. キーフレーム追加
- **目的**: 現在フレームに形状情報（キーフレーム）を記録する。フレーム間の補間の基準点になる。
- **動作**: 現在フレームの形状を `manual` ソースのキーフレームとして保存する。
- **UI**: キーボード `K`、またはプロパティパネルの「追加」ボタン。

### 8-2. キーフレーム削除
- **目的**: 現在フレームのキーフレームを削除する。前後のキーフレームから補間で形状が決まる。
- **UI**: キーボード `Shift+K`、またはプロパティパネルの「削除」ボタン。

### 8-3. キーフレーム複製
- **目的**: 現在フレームの形状を別のフレームにコピーする。
- **UI**: キーボード `Ctrl+D`。

### 8-4. キーフレーム補間
- **目的**: キーフレーム間のフレームでの形状・位置を自動的に滑らかに変化させる。
- **動作**: 2 つのキーフレームの bbox（座標・幅・高さ）と polygon 頂点を線形補間する。補間結果はリアルタイムでプレビューに反映される。
- **補間対象**: 位置 (x, y)、幅 (w)、高さ (h)、多角形頂点 (polygon_points)、回転角度 (rotation)、不透明度 (opacity)。

## 9. マスクスタイル設定

### 9-1. モザイク強度
- **目的**: モザイクのかかり具合（粗さ）を調整する。
- **設定範囲**: 2〜100（数値が低いほど粗い）。
- **UI**: プロパティパネルのスライダー。

### 9-2. 拡張ピクセル（Expand）
- **目的**: マスク形状を外側に膨らませ、対象物がはみ出ないようにする。
- **設定範囲**: 0〜200 px。
- **UI**: プロパティパネルのスピンボックス。

### 9-3. フェザー
- **目的**: マスク境界をぼかし、モザイクとオリジナル映像の境界を滑らかにする。
- **設定範囲**: 0〜50（ピクセル）。
- **UI**: プロパティパネルのスピンボックス。

### 9-4. スタイルのキーフレーム単位上書き
- **目的**: 特定のフレームだけ拡張・フェザーの値を変えたい場合に対応する。
- **動作**: キーフレーム個別の `expand_px` / `feather` を設定できる。`None` の場合はトラックのデフォルト値を使用する。

## 10. AI 検出

### 10-1. 現在フレームで検出
- **目的**: 現在表示中の 1 フレームに対して AI 検出を実行し、結果をトラックとして追加する。
- **動作**: 検出エンジン（DetectorFactory 経由）が選択中のモデルで推論を実行する。結果の bbox を新しいトラックに変換する。
- **UI**: キーボード `Ctrl+Shift+D`。

### 10-2. 選択範囲（In〜Out）で検出
- **目的**: In/Out マーカーで指定したフレーム範囲に対して一括検出を実行する。
- **動作**: サンプリング間隔ごとにフレームをサンプリングし、バックグラウンドスレッド（`DetectionWorker`）で検出を実行する。進捗ダイアログを表示し、ETA を常時更新する。検出完了後、`TrackingService` が複数フレームにわたるトラックを自動生成する。
- **UI**: キーボード `Ctrl+Shift+R`、またはメニューから実行。進捗ダイアログが表示される。キャンセル可能。

### 10-3. 検出結果のトラック自動生成
- **目的**: 複数フレームにわたる検出結果を 1 本の連続するトラックにまとめる。
- **動作**: `TrackingService` が IOU スコア・距離・面積変化を加重スコアで評価し、同一対象として関連付ける。新しいトラックは自動的にタイムラインに追加される。

### 10-4. 輪郭自動抽出
- **目的**: AI 検出の矩形 bbox をより精密な形状（多角形）に変換する。
- **動作**: GrabCut アルゴリズムで前景を抽出し、輪郭を多角形に変換する。失敗時は bbox ベースの矩形多角形にフォールバックする。
- **設定**: 輪郭モード（none / fast / balanced / quality）で精度と速度をトレードオフ。

### 10-5. カテゴリフィルタ
- **目的**: 検出対象のカテゴリを絞り込み、不要な検出結果を除外する。
- **動作**: 選択されたカテゴリ以外の検出結果は無視される。
- **UI**: GPU 設定ダイアログ → カテゴリ選択 / `CategoryFilterDialog`。

---

## 11. 検出設定・GPU 設定

### 11-1. 検出エンジン選択
- **目的**: 使用する AI 検出モデル・エンジンを選択する。
- **選択肢**: NudeNet（軽量）/ NudeNet 640m（高精度）/ EraX v1.1 / YOLOv3 / YOLOv3 Tiny / SSD ResNet34。
- **UI**: GPU 設定ダイアログのドロップダウン。

### 11-2. デバイス選択
- **目的**: 推論を実行するデバイスを選択する。
- **選択肢**: auto（CUDA 優先）/ CUDA 強制 / CPU 強制。
- **UI**: GPU 設定ダイアログ。

### 11-3. 推論解像度設定
- **目的**: 推論時に使用する画像解像度を調整し、速度・精度のバランスを取る。
- **選択肢**: 元解像度 / 1/2 / 1/4 / 640px 固定。

### 11-4. サンプリング間隔設定
- **目的**: 検出実行フレームの間隔を指定する。間隔を広げると処理時間が短縮される。
- **設定範囲**: 1〜100 フレーム。

### 11-5. バッチサイズ設定
- **目的**: 1 回の推論で処理するフレーム数を設定する。GPU の VRAM に合わせて調整する。
- **設定範囲**: 1〜32。VRAM 節約モード時は自動低減。

### 11-6. 信頼度閾値設定
- **目的**: 検出結果を採用する最低スコアを設定する。低すぎると誤検出が増え、高すぎると検出漏れが増える。

### 11-7. 輪郭生成モード設定
- **目的**: 検出後の輪郭抽出アルゴリズムと品質を選択する。
- **選択肢**: none（bbox のみ） / fast（軽量 GrabCut）/ balanced（標準 GrabCut）/ quality（GrabCut + SAM2）。

### 11-8. VRAM 節約モード
- **目的**: VRAM が少ない GPU でも動作するよう、自動的にバッチサイズを下げる。

## 12. モデル管理

### 12-1. モデル取得タブ
- **目的**: 使用可能な AI 検出モデルの一覧を表示し、ダウンロードや変換の案内を行う。
- **表示内容**: モデル名・ライセンス状態・ダウンロード可否・公式リンク。
- **UI**: メニュー「モデル管理」から「モデルの取得」タブ。

### 12-2. 導入済みモデル管理タブ
- **目的**: 実際にシステムに配置されているモデルファイルを一覧表示し、切替・削除ができるようにする。
- **動作**: `models/` ディレクトリ内の `.onnx` ファイルのみを表示する（存在しないファイルは表示しない）。
- **UI**: モデル管理ダイアログの「導入済みモデル」タブ。

### 12-3. EraX モデル変換
- **目的**: PyTorch 形式（`.pt`）の EraX モデルを ONNX 形式に変換し、推論エンジンで使用できるようにする。
- **動作**: アプリ起動時に自動的に `.pt` ファイルを検出して変換を試みる。失敗時はモデル管理ダイアログで手動変換手順を案内する。

## 13. 書き出し（Export）

### 13-1. 書き出しダイアログ
- **目的**: 書き出しの各種設定を行い、ジョブをキューに追加する。
- **UI**: `Ctrl+M` で開く。

### 13-2. 出力解像度設定
- **目的**: 書き出す動画の解像度を指定する。
- **選択肢**: source（元解像度）/ 720p / 1080p / 4K。アスペクト比を保ちながらフィット。

### 13-3. ビデオコーデック設定
- **目的**: 使用する映像圧縮形式を選択する。
- **選択肢**: H.264（h264_nvenc/libx264）/ VP9（libvpx-vp9）。

### 13-4. コンテナ形式設定
- **目的**: 出力ファイルの形式を選択する。
- **選択肢**: mp4 / mov / webm。

### 13-5. ビットレートモード設定
- **目的**: 出力ファイルのビットレート（画質）の制御方式を選択する。
- **選択肢**:
  - **auto**: 解像度に応じた自動設定（720p=8000kbps、1080p=16000kbps、4K=40000kbps）。
  - **manual**: kbps を直接指定。
  - **target_size**: MB でファイルサイズを指定し、ビットレートを逆算。

### 13-6. フレームレート設定
- **目的**: 出力動画の FPS を指定する。
- **選択肢**: source（元の FPS）/ カスタム値。

### 13-7. 音声設定
- **目的**: 音声の処理方式を選択する。
- **選択肢**: copy_if_possible（再エンコードなし）/ encode / none（音声なし）。

### 13-8. GPU 書き出し
- **目的**: GPU（NVENC）を使って高速に書き出す。
- **動作**: CUDA が利用可能な場合 `h264_nvenc` を使用する。利用不可時は CPU（libx264）にフォールバック。

### 13-9. エクスポートプリセット
- **目的**: よく使う書き出し設定を保存・再利用できるようにする。
- **組み込みプリセット**: 6 種（720p / 1080p / 4K / MOV 高品質等）。
- **ユーザー定義プリセット**: 現在の設定を名前を付けて保存・削除できる。

### 13-10. 書き出し前危険フレーム警告
- **目的**: 書き出し結果に問題が生じそうなフレームをあらかじめユーザーに通知する。
- **検出内容**: long_gap（キーフレーム間隔が長い）/ area_jump（マスク面積の急変）/ track_lost（予測補完中のトラックが存在する）。
- **動作**: 危険フレームが検出された場合、警告ダイアログを表示する。ユーザーは「レビュー」「このまま書き出し」「キャンセル」を選択できる。

### 13-11. バックグラウンド書き出し
- **目的**: 書き出し中も UI を操作できるよう、書き出しを非同期で実行する。
- **動作**: `ExportWorker`（QThread）が `ExportService` を実行し、進捗・ETA をリアルタイムで更新する。キャンセル可能。

### 14-1. キュー表示
- **目的**: 書き出しジョブの一覧と状態（待機 / 実行中 / 完了 / キャンセル / エラー）を確認する。

## 15. Undo / Redo（編集履歴）

### 15-1. Undo
- **目的**: 直前の編集操作を取り消す。
- **対象操作**: トラック追加・削除・移動・スタイル変更・キーフレーム追加・削除等、すべての編集操作。
- **UI**: `Ctrl+Z`。再生中は操作不可。

### 15-2. Redo
- **目的**: Undo した操作をやり直す。
- **UI**: `Ctrl+Y` または `Ctrl+Shift+Z`。

### 15-3. 履歴スタック管理
- **動作**: `HistoryService` が `push()` / `undo()` / `redo()` を管理する。各編集操作後に `_commit_history_state()` を呼ぶ必要がある。
- **保存形式**: `Project` オブジェクトの `snapshot()` をスタックに積む（最大 100 段階）。

---

## 16. クラッシュリカバリ

### 16-1. 自動保存
- **目的**: アプリがクラッシュした場合に編集内容を失わないよう、定期的に自動保存する。
- **保存先**: `data/recovery/` ディレクトリ。

### 16-2. 復旧ダイアログ
- **目的**: 起動時に前回の自動保存データが存在する場合、復旧するかどうかを選択させる。
- **選択肢**: 復旧 / 削除して新規開始。
- **UI**: アプリ起動時に `RecoveryDialog` が自動表示される。
