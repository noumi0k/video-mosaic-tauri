# 検出エンジン（モデル）導入手順

> レビュー向け暫定手順（2026-04-13 作成）

---

## 概要

Auto Mosaic の検出モデル（AI モデル）は、本体に同梱していません。  
検出機能を使うには、モデルファイル（`.onnx`）をユーザー自身で入手・配置する必要があります。

モデル未導入でも、アプリの起動・手動マスク追加・編集・書き出しは可能です。

> **注意:** 5月レビュー版の `setup.bat` にはモデル自動取得の処理が含まれていますが、これはレビュー時の利便のための暫定動作です。製品版UIの恒久的な配布導線ではありません。アプリ内での直ダウンロード／ワンクリック取得／ミラー配布は行いません。

---

## モデル未導入時にできること / できないこと

| 操作 | モデル未導入 | モデル導入後 |
|------|:---:|:---:|
| アプリの起動 | OK | OK |
| 動画の読み込み | OK | OK |
| 手動マスク追加（N / Shift+N） | OK | OK |
| マスクの編集・移動 | OK | OK |
| 動画の書き出し | OK | OK |
| 自動検出（全区間 / 区間 / フレーム） | NG | OK |
| モデルの切替 | 表示のみ | OK |

## 導入手順

### 手順 1: モデルファイルの入手

5月レビュー版の `setup.bat` にはモデル自動取得の処理が含まれており、実行時にダウンロードを試みます。成功した場合は `models/` フォルダにモデルファイルが配置されます。

※この自動取得はレビュー時の利便のための暫定動作です。製品版UIでの恒久的な導線ではありません。

自動取得が失敗した場合、または別のモデルを使いたい場合は、配布元のページをブラウザで開き、モデルファイル（`.onnx` 形式）をユーザー自身で取得してください。取得したモデルは以下の手順に従って `models/` フォルダに配置します。

### 手順 2: モデルファイルの配置

入手したモデルファイルを以下のフォルダに配置します：

```
auto-mosaic/
└── models/
    ├── 320n.onnx          ← NudeNet 320n 用
    ├── 640m.onnx          ← NudeNet 640m 用（任意）
    └── erax_v1_1.onnx     ← EraX v1.1 用（任意）
```

#### EraX の `.pt` から `.onnx` への変換について

EraX v1.1 は配布元が `.pt`（PyTorch）のみを提供しているため、`setup.bat` が `ultralytics` 経由で `.onnx` へ自動変換します。

- `setup.bat` 実行時に `ultralytics` があれば `erax_v1_1.onnx` が生成されます
- `ultralytics` が無い場合は `erax_v1_1.pt` のみ配置された状態になります。この状態では EraX 検出は**利用できません**（`EraXAdapter` は `.onnx` のみ読み込みます）
- その場合、「設定 > 検出モデル管理...」ダイアログで EraX が「変換が必要」と表示され、`ultralytics` 導入後に「ONNX に変換」ボタンから変換できます
- この変換は EraX v1.1 専用です。任意の `.pt` の汎用変換には対応しません

### 手順 3: アプリの起動（または再起動）

モデルファイルを配置した後、`run_ui.bat` でアプリを起動（または再起動）してください。  
ツールバーの「検出エンジン」コンボボックス、または「設定 > 検出モデル管理...」ダイアログで、導入したモデル名を選択できます。

### 手順 4: 導入状況の確認

1. 「設定 > 検出モデル管理...」を開く
2. 既知モデル (`NudeNet 320n` / `NudeNet 640m` / `EraX v1.1`) の「利用可否」列を確認
   - `利用可能`: `.onnx` が `models/` に配置済み
   - `変換が必要`: EraX 限定。`erax_v1_1.pt` はあるが `.onnx` が無い状態。`ultralytics` が入っていれば画面下の「ONNX に変換」ボタンが押せます
   - `モデルファイルなし`: 配置が必要。上記手順 1・2 を実施してください
3. ファイルパス列には、アプリが読み込む想定のパスが表示されます

### 手順 5: 検出の実行

1. 動画を開く
2. ツールバーで使用するモデルを確認
3. 「自動検出 ▾」>「全区間検出」をクリック
4. 検出が完了するとトラックが生成されます

## ライセンスに関する注意

各モデルファイルには配布元のライセンスが適用されます。取得・使用の適法性・互換性はユーザー自身で確認してください。Auto Mosaic 本体はモデルファイルのライセンス適法性を保証しません。

---

## 輪郭エンジン（SAM2）について

SAM2 は「輪郭エンジン」であり、検出エンジン（検出モデル）とは別の役割です。  
検出された領域の輪郭精度を向上させるために使用します。

- SAM2 モデル（`sam2_tiny_encoder.onnx`, `sam2_tiny_decoder.onnx`）は OSS の輪郭エンジンとしてアプリに同梱されます（`setup.bat` での取得フロー経由）
- SAM2 は検出エンジン選択 UI には表示されません
- 「詳細設定...」>「輪郭モード」で「高品質 (SAM2 tiny)」を選択すると利用されます

## トラブルシューティング

### 検出ボタンを押しても「検出エンジンが利用できません」と表示される

- `models/` フォルダにモデルファイルが存在するか確認してください
- ファイル名が正しいか確認してください（上記の表を参照）
- アプリを再起動してください

### モデルファイルのダウンロードに失敗する

- ネットワーク接続を確認してください
- プロキシ環境の場合、環境変数 `HTTP_PROXY` / `HTTPS_PROXY` を設定してください
- 手動でモデルファイルをダウンロードし、`models/` に配置してください
