# Auto Mosaic 人間向け仕様書

最終更新: 2026-04-11

## 1. Auto Mosaic とは

Auto Mosaic は、動画の中で隠したい領域にモザイクをかけるためのローカル desktop アプリです。
AI が最初に「ここを隠したほうがよさそう」という候補を作り、ユーザーがそれを確認・修正して、モザイク入りの動画を書き出します。

動画や編集内容はユーザーの PC 内で処理されます。クラウドへ動画を送って解析するアプリではありません。

## 2. 基本の流れ

1. 動画を開く
2. AI 検出を実行する
3. 生成されたマスクトラックを確認する
4. 必要に応じて位置、形、キーフレームを直す
5. プロジェクトを保存する
6. モザイク入り動画を書き出す

## 3. 重要な考え方

### AI は候補を作るだけ

AI の検出結果は最終結果ではありません。
ユーザーが確認し、必要なら手で調整することを前提にしています。

### 編集単位はマスクトラック

Auto Mosaic は、1 フレームごとの孤立した図形ではなく、時間方向につながる「マスクトラック」としてモザイク領域を扱います。
そのため、同じ対象をタイムライン上で追いながら編集できます。

### 手動編集は保護される

ユーザーが手で直したトラックやキーフレームは、AI の再検出で簡単に上書きされないように扱います。
この保護は UI だけの約束ではなく、backend の project state 側でも守る方針です。

### 完全ローカルを前提にする

動画処理、AI 推論、書き出しは PC 内で完結します。
追加モデルの取得など一部の操作ではネットワークを使う可能性がありますが、動画そのものを外部へ送る設計ではありません。

## 4. 現在できること

- 動画を開いて project を作成する
- project を保存、読み込み、別名保存する
- AI でモザイク候補を検出する
- 全体検出、現在フレーム検出、I/O マーカーによる範囲検出を行う
- マスクトラックを作成、削除、複製、分割する
- 楕円と polygon の形を canvas 上で編集する
- キーフレームを追加、移動、複製、削除する
- Undo / Redo を使う
- 60 秒間隔の自動保存を使う
- export 前に危険フレーム警告を確認する
- 解像度、bitrate、音声 mux 設定を指定して動画を書き出す
- 検出、モデル取得、動画読み込み、書き出しなどの長時間処理を Job Panel で確認・キャンセルする
- doctor で Python、ffmpeg、ONNX Runtime、モデルの状態を確認する
- 壊れたモデルや不足モデルを検出前に止める
- review package 用 runtime を準備する

## 5. まだできないこと

優先度付きの詳細 backlog は [../project/unimplemented-features.md](../project/unimplemented-features.md) を正として扱います。
人間向けには、主に次が残っています。

- 前後フレームの形を重ねて見るオニオンスキン表示
- GPU エンコーダの選択 UI と失敗時の CPU fallback
- 複数 export のキュー実行
- Tauri window 上の実操作 E2E テスト
- 教師データ保存
- ローカル再学習
- 正式 installer / updater
- crash recovery dialog

## 6. レビューで見るポイント

- 起動できるか
- 動画を開けるか
- AI 検出が始まり、進捗が見えるか
- 検出結果がタイムラインと canvas に反映されるか
- 手動で直した内容が保存・再読み込み後も残るか
- 書き出し結果にモザイクが入るか
- GPU が使えない環境でも CPU で起動・処理できるか
- エラーや不足モデルが stdout の壊れた文字ではなく UI 上の状態として見えるか

具体的な手順は [../review/review-checklist.md](../review/review-checklist.md) を使います。
